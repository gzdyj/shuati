**问题**: 数据库中文乱码

**解决方案**:
1. 修改 `application.yml` 数据库连接URL：`characterEncoding=utf8`（驱动不支持utf8mb4）
2. 创建 `mysql.cnf` 配置文件强制字符集为utf8mb4
3. 挂载到MySQL容器 `/etc/mysql/conf.d/mysql.cnf`
4. 修复配置文件权限（chmod 644）使其生效

**关键配置**:
```yaml
url: jdbc:mysql://localhost:3307/quiz_master?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
```

## 三、踩坑经验

### 3.1 MySQL驱动字符编码问题

**问题**: `characterEncoding=utf8mb4` 报错 `Unsupported character encoding 'utf8mb4'`

**原因**: MySQL Connector/J 8.x 不支持 utf8mb4 作为 characterEncoding 值

**解决方案**: 使用 `characterEncoding=utf8`，MySQL 会自动升级为 utf8mb4

### 3.2 Docker健康检查失败

**问题**: 使用 `nc` 命令检查后端健康状态失败

**原因**: eclipse-temurin 镜像未安装 netcat

**解决方案**: 安装 curl，使用API端点检查：
```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:8080/api/question/categories"]
```

### 3.3 MySQL配置文件权限问题

**问题**: `mysql.cnf` 挂载后被忽略，提示 `World-writable config file is ignored`

**解决方案**: 执行 `chmod 644 /etc/mysql/conf.d/mysql.cnf` 修复权限

### 3.4 Maven依赖下载缓慢

**问题**: 默认Maven仓库下载依赖超时

**解决方案**: 在 Dockerfile 中配置国内镜像源
```dockerfile
RUN mkdir -p /root/.m2 && echo '<settings><mirrors><mirror><id>aliyunmaven</id><mirrorOf>central</mirrorOf><url>https://maven.aliyun.com/repository/public</url></mirror></mirrors></settings>' > /root/.m2/settings.xml
```

### 3.5 容器名冲突

**问题**: `docker compose up` 提示容器名已被使用

**解决方案**: 先执行 `docker rm -f quizmaster-mysql` 移除旧容器

### 3.6 PowerShell命令语法

**问题**: PowerShell 5 不支持 `&&` 分隔符

**解决方案**: 使用分号 `;` 或单独执行命令
